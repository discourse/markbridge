## See it in action

```ruby
require "markbridge/bbcode"

result = Markbridge.bbcode_to_markdown(
  "[b]Hello[/b] [url=https://example.com]world[/url] [unknown]oops[/unknown]"
)

result.markdown        # => "**Hello** [world](https://example.com) oops"
result.unknown_tags    # => {"unknown" => 2}
result.emissions       # => {} (custom Tags can populate this for the importer)
```

`*_to_markdown` returns a `Conversion` value object — the rendered string is on `.markdown`, side data and unknown tags ride alongside.

  <Card title="Forum migrations, first-class" icon="rocket">

  </Card>
  <Card title="Four input formats" icon="random">

  </Card>
  <Card title="Graceful degradation" icon="approve-check-circle">
    Unknown tags don't crash — they're tracked and surfaced via `Conversion#unknown_tags` so a migration can audit, drop, or fail per its own policy. Malformed input degrades to plain text instead of raising.
  </Card>
  <Card title="Extensible without monkey-patching" icon="puzzle">
    Register custom handlers to recognize new tags. `HandlerRegistry#overlay` wraps the default for delegation. Build a custom `Renderer` once with `Markbridge.discourse_renderer(...)` and reuse across thousands of posts.
  </Card>
  <Card title="Performance-conscious" icon="setting">
    Single-pass parsing, O(1) parent lookups during render, minimal allocations on the hot paths. Linear in input size. Reusable Renderer with zero overhead on the no-emit path.
  </Card>
  <Card title="Battle-tested" icon="star">
    Powers Discourse forum migrations. Extensive unit, integration, and end-to-end system tests keep real-world inputs covered.
  </Card>
